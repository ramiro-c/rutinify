Plan de Desarrollo Iterativo:

**Fase 1: MVP (Producto Mínimo Viable) - Funcionalidad Central**

1.  **Estructura del Proyecto:**
    *   Inicializar un proyecto React (usando Vite para mayor rapidez).
    *   Elegir una librería de componentes como **Material-UI** o **Chakra UI** para tener una interfaz limpia y funcional desde el inicio.

2.  **Gestión de Rutinas (CRUD):**
    *   **Crear/Editar Rutinas:** Un formulario intuitivo donde puedas nombrar una rutina, definir los días (Día 1, Día 2, etc.) y añadir ejercicios a cada día.
    *   **Añadir Ejercicios:** Dentro de cada día, un buscador o selector para agregar ejercicios. Para cada ejercicio, definiremos los campos: `series`, `repeticiones objetivo`, `tempo` y `descanso`.
    *   **Ver Rutinas:** Una vista principal que liste todas tus rutinas guardadas.

3.  **Almacenamiento Local:**
    *   Toda la información (rutinas, ejercicios, etc.) se guardará en `localStorage`. Definiremos una estructura de datos JSON clara desde el principio para que sea fácil de exportar.

**Fase 2: Ejecución y Tracking del Entrenamiento**

1.  **Modo "Entrenamiento":**
    *   Al seleccionar una rutina, entrarás en un modo "en vivo".
    *   La interfaz te mostrará el ejercicio actual, con campos para que anotes el **peso** y las **repeticiones** realizadas en cada serie.
    *   Un cronómetro podría ser útil para medir los descansos.

2.  **Historial de Progreso:**
    *   Guardaremos un registro de cada sesión de entrenamiento completada. Esto es clave para luego analizar tu progreso.

**Fase 3: Importación y Exportación de Datos**

1.  **Exportación a CSV:**
    *   Crearemos una función para convertir tus rutinas y tu historial de progreso a formato CSV. Con un solo clic, podrás descargar un archivo compatible con Excel o Google Sheets.

2.  **Importación desde CSV:**
    *   Permitiremos subir un archivo CSV para crear o actualizar rutinas. Definiremos una plantilla CSV clara para que sepas qué formato usar. Esto es perfecto para si quieres planificar tus rutinas en una hoja de cálculo y luego subirlas a la app.

**Fase 4: Integración con Google Sheets (Futuro)**

*   Una vez que todo lo anterior funcione, abordaremos la API de Google Sheets. Esto es más complejo porque requiere autenticación (que el usuario inicie sesión con su cuenta de Google), pero permitiría una sincronización más directa.
